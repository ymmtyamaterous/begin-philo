# 全体概要仕様書

## 1. プロジェクト概要

### サービス名
**bphilo（ビフィロ）** — 入門者のための哲学学習サイト

### コンセプト
- 教養として哲学を学びたい社会人・大学生を対象とした、完全無料の哲学入門学習プラットフォーム
- 「難しい言葉は後回しでいい。まず、一緒に考えよう。」という姿勢でアクセシブルなコンテンツを提供
- ソクラテスからニーチェまで、東洋から西洋まで、幅広い哲学思想をわかりやすく解説

### ターゲットユーザー
| セグメント | 説明 |
|-----------|------|
| 社会人入門者 | 哲学に興味はあるが、どこから始めれば良いかわからない社会人 |
| 大学生 | 教養・レポート・授業の補助として哲学を学びたい大学生 |
| 再学習者 | かつて哲学を学んだが、もう一度体系的に整理したい人 |

| 用語集 | 哲学専門用語をわかりやすく解説 | 中 |
| 検索 | 哲学者・テーマ・記事・用語のキーワード検索 | 高 |

## 4. モノレポ構成

```
better-t-app/
├── apps/
│   ├── web/          # フロントエンド (React + TanStack Router + Vite)
│   └── server/       # バックエンド (Hono + oRPC)
└── packages/
    ├── api/          # oRPC ルーター定義
    ├── auth/         # better-auth 設定
    ├── db/           # Drizzle ORM スキーマ・クライアント
    ├── env/          # 環境変数バリデーション
    ├── config/       # 共通 tsconfig 等
    └── ui/           # 共通UIコンポーネント (shadcn/ui)
```

## 6. 非機能要件

### パフォーマンス
- 初回ロード（LCP）: 2.5秒以内
- TanStack Query によるデータキャッシュでレスポンシブな操作感
- コンテンツは静的に近い構造のため、積極的にキャッシュ活用

### アクセシビリティ
- セマンティックHTMLの使用
- キーボードナビゲーション対応
- 適切なARIAラベル設定

### レスポンシブ対応
- モバイル（〜640px）・タブレット（641px〜900px）・デスクトップ（901px〜）の3ブレークポイント対応

### セキュリティ
- better-auth によるセッション管理
- 認証が必要なルートはサーバー・クライアント双方でガード
- SQLインジェクション対策（Drizzle ORM のパラメータ化クエリ）

### コンテンツ管理
- 初期リリースは Drizzle seed でコンテンツデータを投入
- 将来的には管理画面の追加を検討
